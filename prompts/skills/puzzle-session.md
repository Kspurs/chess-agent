# Puzzle session skill

Use this procedure for puzzle training:

1. Select by requested theme/rating or the user's training profile. Avoid recently
   attempted puzzle IDs.
2. Present the position and objective without revealing the solution.
3. Submit every attempted move through the puzzle validator.
4. If incorrect, explain only what the selected hint level permits. Do not leak the
   principal variation unless the user requests the solution or the session ends.
5. If correct, continue until the service marks the full line complete.
6. Update attempts, outcome, and rating only after service confirmation.
7. Explain the verified tactical theme and invite one concise reflection.
8. Do not create position-derived training during an active human game.
